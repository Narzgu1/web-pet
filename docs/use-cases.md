# Use Cases (Сценарии использования)

## UC-1: Просмотр форума

**Актор:** Пользователь

**Предусловия:** Пользователь находится на главной странице

**Основной поток:**
1. Пользователь нажимает на карточку "Forum"
2. Система отображает список обсуждений
3. Пользователь просматривает посты
4. Пользователь может открыть категории и события в боковой панели

**Постусловия:** Пользователь видит список обсуждений

---

## UC-2: Создание нового обсуждения

**Актор:** Пользователь

**Предусловия:** Пользователь находится на странице форума

**Основной поток:**
1. Пользователь нажимает кнопку "Start New Discussion"
2. Система открывает модальное окно
3. Пользователь заполняет заголовок, выбирает категорию, вводит текст
4. Пользователь нажимает "Post"
5. Система показывает сообщение об успешной публикации
6. Модальное окно закрывается

**Альтернативный поток:**
- Пользователь может нажать "Cancel" для отмены

---

## UC-3: Просмотр профиля питомца

**Актор:** Пользователь

**Предусловия:** Пользователь авторизован и имеет питомцев

**Основной поток:**
1. Пользователь переходит в раздел "Pet Profile"
2. Система отображает список питомцев в боковой панели
3. Пользователь выбирает питомца
4. Система отображает детальную информацию о питомце
5. Пользователь может изменить статус вакцинации через переключатель

---

## UC-4: Сообщение о потерянном питомце

**Актор:** Пользователь

**Предусловия:** Пользователь находится на странице Lost & Found

**Основной поток:**
1. Пользователь заполняет форму: имя, локация, описание
2. Пользователь нажимает "Submit Report"
3. Система валидирует данные
4. Система сохраняет объявление
5. Объявление появляется в списке

---

## UC-5: Поиск заведений на карте

**Актор:** Пользователь

**Предусловия:** Пользователь находится на странице Facilities Map

**Основной поток:**
1. Пользователь видит карту с маркерами
2. Пользователь применяет фильтры (клиники, магазины, парки)
3. Система обновляет отображаемые маркеры
4. Пользователь кликает на маркер
5. Система показывает информацию о заведении в боковой панели
6. Пользователь может получить маршрут

---

## UC-6: Администрирование контента

**Актор:** Администратор

**Предусловия:** Администратор авторизован

**Основной поток:**
1. Администратор переходит в Admin Panel
2. Система отображает панель управления
3. Администратор выбирает действие (модерация постов, управление заведениями, просмотр жалоб)
4. Система отображает соответствующий интерфейс
5. Администратор выполняет действия (одобрение, удаление, редактирование)
